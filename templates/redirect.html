Understanding Open Redirect Vulnerabilities

Open Redirect vulnerabilities occur when an application takes a user-supplied URL and redirects the user to that URL without proper validation. Attackers can exploit this to redirect users to malicious sites.

Example: An attacker might use a URL like example.com/redirect?url=http://malic-site.com to redirect users to an external website.

To prevent open redirects, applications should validate and sanitize user inputs, ensuring only trusted URLs are allowed for redirection.

  • http://
  • https://
  • Protocol-relative URL
  • Another Protocol-relative URL

Use this information wisely and only test for vulnerabilities on systems you have explicit permission to test.
